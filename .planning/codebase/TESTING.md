## Test Structure

**Suite Organization:**
```typescript
describe('AppComponent', () => {
  beforeEach(async () => {
    await TestBed.configureTestingModule({
      imports: [RouterTestingModule],
      declarations: [AppComponent],
    }).compileComponents();
  });

  it('should create the app', () => {
    const fixture = TestBed.createComponent(AppComponent);
    const app = fixture.componentInstance;
    expect(app).toBeTruthy();
  });

  it(`should have as title 'token-manager-angular'`, () => {
    const fixture = TestBed.createComponent(AppComponent);
    const app = fixture.componentInstance;
    expect(app.title).toEqual('token-manager-angular');
  });
});
```

**Patterns:**
- `beforeEach(async () => { ... })`: Configure TestBed module and dependencies before each test
- `TestBed.configureTestingModule()`: Set up testing module with imports/declarations
- `.compileComponents()`: Compile component template and CSS (required for external templates)
- `TestBed.createComponent()`: Create component fixture for testing
- `fixture.componentInstance`: Access component class instance
- `fixture.detectChanges()`: Trigger change detection and lifecycle hooks
- `fixture.nativeElement`: Access native DOM element

## Mocking

**Framework:** Jasmine spies (built-in to Jasmine)

**Current approach:**
- RouterTestingModule used to mock routing for AppComponent
- No service mocking detected in existing tests

**What to Mock:**
- External services (GitHubService, FigmaService, FileService)
- HTTP requests (HttpClient)
- Router/Navigation
- Browser APIs that are environment-specific

**What NOT to Mock:**
- Angular core services (if possible to use real instances)
- Local service dependencies that are part of the same test scope
- Type definitions and interfaces
- Utility functions (these are pure and testable as-is)

## Test Types

**Unit Tests:**
- Scope: Individual functions/methods in isolation
- Approach: Mock external dependencies, test single responsibility
- Example from `app.component.spec.ts`:
  ```typescript
  it('should create the app', () => {
    const fixture = TestBed.createComponent(AppComponent);
    const app = fixture.componentInstance;
    expect(app).toBeTruthy();
  });
  ```

**Integration Tests:**
- Scope: Multiple services working together
- Approach: Use TestBed to configure real service dependencies
- Not found in current codebase
- Should test: Token import flow, GitHub API integration with TokenService, File export pipeline

**E2E Tests:**
- Framework: Not currently used (would be Protractor/Cypress/Playwright)
- Scope: End-to-end user workflows
- Not implemented: No E2E test suite exists

## Common Patterns

**Async Testing:**
```typescript
// From existing app.component.spec.ts
beforeEach(async () => {
  await TestBed.configureTestingModule({
    imports: [RouterTestingModule],
    declarations: [AppComponent],
  }).compileComponents();
});

// For async operations in tests:
it('should handle async operations', async () => {
  const result = await someAsyncFunction();
  expect(result).toBeDefined();
});

// Alternative with done callback:
it('should complete async work', (done) => {
  someAsyncFunction().then(() => {
    expect(true).toBe(true);
    done();
  });
});
```

**Error Testing:**
Not found in existing tests. Recommended pattern:
```typescript
it('should handle validation errors', () => {
  const result = validateGitHubToken('invalid');
  expect(result.isValid).toBe(false);
  expect(result.errors.length).toBeGreaterThan(0);
  expect(result.errors[0]).toContain('Invalid GitHub token format');
});

it('should throw on API error', async () => {
  const service = TestBed.inject(GitHubService);
  spyOn(service, 'validateConfig').and.throwError('Network error');

  expect(() => service.validateConfig(mockConfig))
    .toThrowError('Network error');
});
```

**Template Testing:**
```typescript
// From app.component.spec.ts
it('should render title', () => {
  const fixture = TestBed.createComponent(AppComponent);
  fixture.detectChanges();
  const compiled = fixture.nativeElement as HTMLElement;
  expect(compiled.querySelector('.content span')?.textContent)
    .toContain('token-manager-angular app is running!');
});
```
