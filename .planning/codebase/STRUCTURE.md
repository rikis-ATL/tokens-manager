## Directory Purposes

**src/app:**
- Purpose: Next.js App Router routes and pages
- Contains: Server and client components for UI and API endpoints
- Generated by: Next.js during development/build

**src/app/api:**
- Purpose: Backend API endpoints (server-side route handlers)
- Contains: Next.js route handlers responding to HTTP requests
- Convention: Each `route.ts` exports GET, POST, PUT, DELETE, PATCH methods

**src/app/api/tokens:**
- Purpose: Token CRUD operations
- Contains:
  - `route.ts`: GET all tokens, flattened and grouped
  - `[...path]/route.ts`: PUT to update specific token value

**src/app/api/import:**
- Purpose: Import tokens from external sources
- Contains: `github/route.ts` - recursive GitHub directory import endpoint

**src/app/api/export:**
- Purpose: Export tokens to external targets
- Contains: `github/route.ts`, `figma/route.ts` for GitHub PR and Figma exports

**src/components:**
- Purpose: Reusable React client components
- Contains:
  - `TokenTable.tsx`: Display and edit tokens in tabular format
  - `TokenGeneratorForm.tsx`, `TokenGeneratorFormNew.tsx`: Create new tokens
  - `GitHubConfig.tsx`: Configure GitHub credentials
  - `GitHubDirectoryPicker.tsx`: Browse GitHub repository structure
  - `ToastNotification.tsx`, `LoadingIndicator.tsx`: UI feedback components
  - `JsonPreviewDialog.tsx`: Preview JSON structures
  - `TokenGeneratorDocs.tsx`: Documentation component

**src/services:**
- Purpose: Encapsulate business logic and external service integrations
- Contains:
  - `token.service.ts`: Token processing, structure detection, reference resolution (core engine)
  - `github.service.ts`: GitHub API operations
  - `figma.service.ts`: Figma API operations
  - `file.service.ts`: File import/export operations
  - `index.ts`: Barrel export of all services

**src/types:**
- Purpose: TypeScript type definitions and interfaces
- Contains:
  - `token.types.ts`: All token-related types (Token, TokenType, TokenGroup, GitHub types, Figma types, etc.)
  - `index.ts`: Barrel export of all types

**src/utils:**
- Purpose: Utility functions and helper classes
- Contains:
  - `tokenUpdater.ts`: TokenUpdater class for filesystem token operations
  - `token.utils.ts`: Token manipulation utilities
  - `ui.utils.ts`: UI-related utility functions
  - `validation.utils.ts`: Token validation functions
  - `index.ts`: Barrel export of utilities

**tokens/ (symlink):**
- Purpose: Design token JSON files
- Contains: Hierarchical JSON token definitions
- Link target: `../design-tokens/tokens`

**token-manager-{variant}:**
- Purpose: Alternative implementations (Stencil Web Components, Angular, Vite variants)
- Workspace members: Defined in root package.json workspaces

## Naming Conventions

**Files:**
- React components: PascalCase with `.tsx` extension (e.g., `TokenTable.tsx`, `GitHubConfig.tsx`)
- Services: camelCase with `.service.ts` extension (e.g., `token.service.ts`, `github.service.ts`)
- Utilities: camelCase with `.utils.ts` extension (e.g., `token.utils.ts`, `validation.utils.ts`)
- API routes: `route.ts` in directory matching the route path (e.g., `src/app/api/tokens/route.ts`)
- Types: camelCase with `.types.ts` extension (e.g., `token.types.ts`)
- Barrel exports: `index.ts` (e.g., `src/components/index.ts`)

**Directories:**
- Page routes: kebab-case for multi-word paths (e.g., `/generate` → `src/app/generate/`)
- Dynamic routes: square brackets for variables (e.g., `[...path]` for catch-all, `[id]` for single param)
- API paths: match URL structure exactly (e.g., `/api/import/github` → `src/app/api/import/github/route.ts`)

**Functions:**
- camelCase for all function names
- Services: methods that perform actions (fetch, create, update, delete)
- Private methods: prefixed with underscore in service classes (e.g., `_detectStructureType()`)

**Variables:**
- camelCase for all variables and parameters
- Constants: UPPER_SNAKE_CASE (e.g., `TOKEN_TYPES` in token.types.ts)
- React state: `const [state, setState] = useState(...)` pattern

**Types:**
- PascalCase for interface names (e.g., `TokenGroup`, `GitHubService`, `FileImportResult`)
- SCREAMING_SNAKE_CASE for exported type arrays (e.g., `TOKEN_TYPES`)
- Generic types: PascalCase (e.g., `<T>`)

## Where to Add New Code

**Clean code rule:** Place logic in the correct layer (see `CLEAN-CODE.md`). Extract to utils/services when functions exceed ~30 lines or components exceed ~300 lines.

**New Feature (Token Processing Enhancement):**
- Primary logic: `src/services/token.service.ts` - Add methods to TokenService class
- Filesystem operations: `src/utils/tokenUpdater.ts` - Add helper methods
- Tests (if added): Create `src/services/token.service.test.ts`

**New Component (UI Feature):**
- Implementation: `src/components/{FeatureName}.tsx` - Create new component file
- Styling: Use Tailwind CSS classes (already configured)
- State management: Use React hooks (useState, useEffect)
- Tests (if added): Create `src/components/{FeatureName}.test.tsx`

**New API Endpoint:**
- Route: Create directory structure matching URL path in `src/app/api/`
- Handler: Create `src/app/api/{path}/route.ts` with GET/POST/PUT/DELETE methods
- Logic: Delegate to services in `src/services/`
- Error handling: Use NextResponse with appropriate status codes

**New Service (External Integration):**
- Implementation: Create `src/services/{name}.service.ts`
- Types: Add interfaces to `src/types/token.types.ts` or create separate file
- Export: Add to `src/services/index.ts` barrel export

**Utilities:**
- Helpers: Add to `src/utils/{category}.utils.ts`
- Classes: Create new file `src/utils/{Name}.ts` (e.g., `TokenUpdater`)
- Export: Add to `src/utils/index.ts` barrel export

**Types:**
- Add all new interfaces to `src/types/token.types.ts`
- Group by feature (Token types, GitHub types, Figma types, etc.)
