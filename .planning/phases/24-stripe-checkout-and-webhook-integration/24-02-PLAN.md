|----------|-------------|
| browser → /api/stripe/checkout | Authenticated user sends priceId — server validates against env allow-list |
| browser → /api/stripe/portal | Authenticated user triggers portal session — server uses org-bound stripeCustomerId |
| server → checkout.stripe.com | Session creation with server-trusted organizationId and priceId in metadata |
| server → billing.stripe.com | Portal session with server-resolved stripeCustomerId |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-24-03 | Tampering | /api/stripe/checkout priceId | MEDIUM | mitigate | `priceIdToTier(priceId) === null → 400 Invalid priceId` — server allow-list rejects any priceId not matching STRIPE_PRO_PRICE_ID or STRIPE_TEAM_PRICE_ID env vars. Prevents subscribing the org to an attacker-chosen plan. Acceptance: test case 5 in checkout-route.test.ts asserts 400 and session.create NOT called. |
| T-24-04 | Elevation of Privilege | /api/stripe/portal cross-org access | HIGH | mitigate | Portal session uses org.stripeCustomerId resolved from `requireAuth().user.organizationId` — NEVER from request body. Attacker cannot pass another org's stripeCustomerId. Acceptance: portal-route.test.ts test 5 asserts `stripe.billingPortal.sessions.create` called with customer resolved from DB lookup of authed user's org. |
| T-24-05 | Spoofing | metadata.organizationId in checkout session | MEDIUM | mitigate | Server writes `metadata.organizationId` from `auth.user.organizationId` (trusted session claim). Request body is NOT read for organizationId. Webhook handler (Plan 03) cross-references `session.customer` (Stripe-assigned) as the primary lookup key. Acceptance: checkout-route.test.ts test 8 asserts metadata.organizationId = server session value. |
| T-24-06 | Information Disclosure | CSRF to create unwanted subscriptions | MEDIUM | mitigate | `requireAuth()` validates NextAuth session cookie (SameSite=lax by default); Next.js same-origin fetch from same-app /upgrade page is sufficient for ASVS L1. | T-24-07 | Denial of Service | SELF_HOSTED Stripe SDK init | MEDIUM | mitigate | SELF_HOSTED check is FIRST statement in each route — before requireAuth, before DB, before getStripe(). Acceptance: test 1 in both route tests asserts requireAuth NOT called when SELF_HOSTED=true. |
| T-24-08 | Information Disclosure | NEXT_PUBLIC_STRIPE_*_PRICE_ID exposure | LOW | accept | Price IDs are public-safe (not secrets — Stripe docs confirm). Server-side allow-list validation means exposing them client-side cannot be weaponized. |
