<objective>
Close 4 gaps found during Phase 24 UAT:

1. **/account link missing from navigation** (Test 8 — minor): /account page exists but is only reachable via direct URL. Add it to UserMenu dropdown (alongside Settings) and OrgSidebar nav (alongside Settings/Users).

2. **Collection limit check fires too late** (Test 3 — major): clicking "New Collection" opens the create dialog before any limit is checked; user fills the form then sees the modal only after submit. Fix: pre-check the collection limit on click, skip the dialog entirely if at limit. The UAT gap text says "navigate directly to /upgrade" but this is superseded by the role-based gap (also Test 3) which explicitly specifies: admin at limit → /account; non-admin at limit → UpgradeModal with "contact your admin" CTA. The UpgradeModal path is the correct non-admin behavior — it keeps users in-context and avoids sending non-admins to the checkout flow they cannot complete alone.

3. **UpgradeModal shows `current: 0` instead of real usage** (Test 3 — major): the subtitle reads "You're currently using 0" because the modal renders directly from the 402 payload without fetching actual counts. Fix: fetch GET /api/org/usage on modal open to display real collection and token counts.

4. **Role-based upgrade behavior** (Test 3 — major): admins should not see the UpgradeModal — they should be navigated directly to /account where they can manage their plan. Non-admins should see the UpgradeModal with a "Contact your admin to upgrade" message instead of the "View Plans" checkout CTA. The provider (UpgradeModalProvider) handles admin routing; the modal always renders for non-admins only and always shows the non-admin CTA (no `isAdmin` prop needed on the modal itself — the provider guarantees only non-admins reach it).

All 4 gaps are independent and execute in a single wave. No new pages, no new API routes, no schema changes.
</objective>
