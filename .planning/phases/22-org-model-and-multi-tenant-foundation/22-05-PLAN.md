<objective>
Wire `assertOrgOwnership()` into all 17 collection-scoped route handlers (D-08), completing the API-layer enforcement required by TENANT-01.

Plan 02 created the `assertOrgOwnership()` function. This plan wires it into every route that handles a specific collection. Without this plan, any authenticated user can access any collection by guessing its ObjectId — the enforcement primitive exists but is never called.

**Pattern for all by-id routes (routes 1-13):**
```typescript
const session = await requireAuth();           // existing
if (session instanceof NextResponse) return session;  // existing
const guard = await assertOrgOwnership(session, params.id);  // NEW
if (guard) return guard;                       // NEW
// ... existing business logic unchanged ...
```

**Pattern for body-id routes (routes 14-17):**
```typescript
const session = await requireAuth();
if (session instanceof NextResponse) return session;
const body = await req.json();
const collectionId = body.mongoCollectionId ?? body.collectionId ?? '';
const guard = await assertOrgOwnership(session, collectionId);
if (guard) return guard;
// ... existing business logic unchanged ...
```

Export routes pass collectionId in the request body rather than URL params (RESEARCH.md). The guard is conditional on the field being present — routes where collectionId is optional skip the guard when absent (safe: no collection = no access needed).
</objective>
