```typescript
export interface ExpressionContext {
  resolveTokenReference?: (ref: string) => string;
  a?: number;
}

```

Behavior (from source review):
- Returns `null` on any parse/evaluation error (try/catch swallows errors internally)
- Strips outer `calc(...)` wrapper (case-insensitive)
- Substitutes `{token.path}` refs via `ctx.resolveTokenReference`; returns null if any ref cannot resolve to a number
- `coerceToNumber` parses leading numeric (handles `16px`, `1.5rem`)
- Supports `+`, `-`, `*`, `/`, parentheses, unary `-`/`+`, variable `a`, decimal literals
- Throws "Division by zero" when divisor is 0
- Throws "Variable a is not bound" when `a` is referenced but `ctx.a` is undefined
- Unit suffix on literals is consumed: `5px * 2` parses as `5 * 2 = 10`
