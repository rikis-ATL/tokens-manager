  <action>
    Add a new exported function `validateExpression(raw: string, ctx?: ExpressionContext): string | null` to `src/lib/mathExpression.ts`. It MUST:
    - Return `null` when the expression is valid (would evaluate to a finite number)
    - Return `null` when `raw.trim() === ''` (empty is not an error state for validation; the UI treats empty as "no expression")
    - Return the thrown error message string when parsing fails (e.g. 'Variable a is not bound', 'Division by zero', 'Missing closing parenthesis', 'Unexpected token at position N: "..."', 'Unexpected end of expression', 'Unexpected character at position N: "..."')
    - Return the string 'Unresolved token reference' when `substituteRefs` returns null (no resolver or non-numeric resolution) — detect this by substituting first; if null, return this exact string
    - Return 'Invalid expression' as a fallback when a non-Error is thrown or result is non-finite

    Implementation MUST reuse the existing private helpers `stripCalc`, `substituteRefs`, `parseExpr` (do not duplicate parser logic). Refactor minimally if needed to export them privately within the module.

    Example shape:
    ```typescript
    export function validateExpression(raw: string, ctx?: ExpressionContext): string | null {
      const expr = raw.trim();
      if (!expr) return null;
      try {
        const stripped = stripCalc(expr);
        const substituted = substituteRefs(stripped, ctx?.resolveTokenReference);
        if (substituted === null) return 'Unresolved token reference';
        const result = parseExpr(substituted, ctx?.a);
        if (!isFinite(result)) return 'Invalid expression';
        return null;
      } catch (err) {
        return err instanceof Error ? err.message : 'Invalid expression';
      }
    }
    ```

    Do NOT modify `evaluateExpression` signature or behavior (the test suite in plan 01 depends on it).
  </action>
