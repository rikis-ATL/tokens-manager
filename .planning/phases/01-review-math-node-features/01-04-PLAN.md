<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Browser verification of Expression mode</name>
  <what-built>
    - Math node Expression mode toggle (already in code)
    - `{token.path}` resolution inside expressions (via resolveTokenReference plumbing in plan 03)
    - Red border + inline error message on invalid expression after blur (plan 02)
    - Full unit test coverage for evaluateExpression (plan 01)
  </what-built>
  <how-to-verify>
    1. Start the dev server: `yarn dev`. Open http://localhost:3000
    2. Navigate to any collection → /collections/[id]/tokens
    3. Select a token group; open the graph panel
    4. Add a Math node; click the `Expr` tab to switch to Expression mode
    5. Wire a numeric input into the `a` handle (e.g. from a Constant node set to 10)
    6. CASE A — Valid expression without refs: type `a * 2 + 3` into the textarea, blur.
       - Expected: no red border, hint text visible, Result preview shows `23`
    7. CASE B — Valid expression with token ref: ensure the collection has at least one numeric token (e.g. `spacing.base = 16px`); type `a + {spacing.base}`, blur.
       - Expected: no red border, Result preview shows `26`
    8. CASE C — Invalid syntax: type `a * ` (trailing operator), blur.
       - Expected: RED border on textarea, error message below reads something like "Unexpected end of expression" in red
    9. CASE D — Unbound variable: clear the wire into `a`, type `a + 1`, blur.
       - Expected: RED border, message "Variable a is not bound"
    10. CASE E — Unresolved ref (node UI): type `{does.not.exist}`, blur.
        - Expected: RED border, message "Unresolved token reference"
    11. CASE F — Recovery: fix CASE C to `a * 2`, blur.
        - Expected: red border clears, hint text returns
    12. CASE G — No flashing on keystroke: type slowly `a +`; while typing, border MUST stay default (not red). Only on blur does it evaluate.
    13. CASE H — Persistence: reload page; the expression mode + formula persists on the node
    14. CASE I — Path extraction: if the node feeds a TokenOutput/Json node that emits token paths, verify the theme↔default path-mismatch indicator no longer shows spurious mismatches caused by null-valued expression nodes

    Record PASS/FAIL + notes for each case in `.planning/phases/01-review-math-node-features/01-04-UAT.md` under a `## Browser Verification` section with a checklist.
  </how-to-verify>
  <resume-signal>Type "approved" if all cases pass, or describe failing cases to trigger gap-closure planning.</resume-signal>
</task>
