  <how-to-verify>
    Before verifying, ensure the dev server is running: `npm run dev`

    **Scenario 1 — Collection with themes: export theme selector**
    1. Open a collection that has at least one theme. Go to /collections/[id]/config.
    2. Confirm "Export theme:" label + selector appears above the Build Output panel.
    3. Confirm the selector shows "Collection default" as default value plus each theme name.

    **Scenario 2 — Collection with no themes: selector is hidden**
    1. Open a collection that has zero themes. Go to /collections/[id]/config.
    2. Confirm NO "Export theme:" selector appears (selector is hidden when themes.length === 0).

    **Scenario 3 — SD build with Collection default (no comment)**
    1. On a collection with themes, keep the selector at "Collection default".
    2. Click "Build Tokens". Wait for build to complete.
    3. Switch to CSS tab. Confirm there is NO `/* Generated by Tokens Manager, theme: ... */` comment at the top.

    **Scenario 4 — SD build with a theme selected (comment header injected)**
    1. On a collection with at least one theme, select a theme from the "Export theme:" selector.
    2. Confirm the build auto-triggers (the panel shows Loading, then result).
    3. Switch to CSS tab. Confirm the FIRST line is `/* Generated by Tokens Manager, theme: [theme name] */`.
    4. Switch to JSON tab. Confirm there is NO comment at the top of JSON output.
    5. Optionally switch to SCSS or JS tabs and confirm they also have the comment header.

    **Scenario 5 — SD build merges theme tokens (Enabled groups use theme values)**
    1. If you have a theme with at least one token value overridden (different from collection default), select that theme.
    2. Click Build Tokens (or wait for auto-build).
    3. Check the CSS output for a token that was overridden — its value should reflect the theme's value, not the default.
    (This scenario is optional if no token overrides exist in test data.)

    **Scenario 6 — Figma Export dialog Enterprise note**
    1. Open any collection's Config page that has Figma configured (or find the Figma export button).
    2. Click the "Export to Figma" button to open the dialog.
    3. Confirm "Figma Variables API requires Enterprise plan." text is visible near the Export button.
    4. Confirm the Export button is still active (not disabled by the note).
  </how-to-verify>
