Migrate `FigmaConfig.tsx`. The component uses `isOpen` boolean state to show/hide the modal.

**Pattern:** Add `const dialogRef = useRef<HTMLElement>(null)` and a useEffect to sync `isOpen` → dialog open/close:
```typescript
const dialogRef = useRef<HTMLElement>(null);
useEffect(() => {
  const el = dialogRef.current as any;
  if (!el) return;
  if (isOpen) { el.openDialog?.(); } else { el.closeDialog?.(); }
}, [isOpen]);
```

**Trigger button** (always visible in header):
```tsx
<at-button
  label={isConnected ? '' : 'Configure Figma'}
  onAtuiClick={() => setIsOpen(true)}

    isConnected ? 'bg-green-100 text-green-800 hover:bg-green-200' : 'bg-gray-100 text-gray-700 hover:bg-gray-200'
  }`}
>
  {isConnected ? (
    <span className="flex items-center gap-2">
      <span className="w-2 h-2 bg-green-500 rounded-full"></span>
      <span className="max-w-[120px] truncate">{config.fileKey}</span>
    </span>
  ) : null}
</at-button>
```

Note: If `at-button` does not support children for the connected indicator, pass label as empty string and use a wrapping div with the status indicator as children content — or keep the outer div structure with a plain button just for the trigger (acceptable if at-button children rendering is unclear). Prefer at-button; fall back to keeping the trigger as a plain HTML button only if the label+children combination doesn't work with the current ATUI version.

**Modal** — replace the `{isOpen && (...)}` block with an `at-dialog` that is always rendered but controlled via ref:

Replace:
