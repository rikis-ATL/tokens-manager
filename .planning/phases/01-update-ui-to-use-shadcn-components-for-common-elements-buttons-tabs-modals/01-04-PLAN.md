**SaveCollectionDialog.tsx:**

The component receives `isOpen: boolean` prop — synchronise the at-dialog open/close state with this prop using a useEffect.

Add `import React, { useState, useEffect, useRef } from 'react';` (already has these).
Add `const dialogRef = useRef<HTMLElement>(null);`

Add a useEffect that opens/closes the dialog when `isOpen` changes:
```typescript
useEffect(() => {
  if (!dialogRef.current) return;
  const el = dialogRef.current as any;
  if (isOpen) {
    el.openDialog?.();
  } else {
    el.closeDialog?.();
  }
}, [isOpen]);
```

Replace the outer `<div className="fixed inset-0 z-50 ...">` wrapper and inner `<div className="bg-white rounded-lg ...">` with `<at-dialog ref={dialogRef} backdrop={true} close_backdrop={false}>` + inner div.

Replace the `if (!isOpen) return null;` guard — keep it as the at-dialog handles visibility, but the early return can remain to avoid rendering the at-dialog content at all when closed (simpler).

Replace all `<button>` elements:
- Close (✕) button → `<at-button label="✕" onAtuiClick={onCancel} disabled={isSaving} className="text-gray-500 hover:text-gray-700" />`
- Cancel button → `<at-button label="Cancel" onAtuiClick={onCancel} disabled={isSaving} className="px-4 py-2 text-sm font-medium text-gray-700 bg-white border border-gray-300 rounded-md hover:bg-gray-50" />`
- Save button → `<at-button label={isSaving ? 'Saving...' : 'Save'} onAtuiClick={handleSave} disabled={isSaving || !name.trim()} className="px-4 py-2 text-sm font-medium text-white bg-blue-600 rounded-md hover:bg-blue-700" />`
- Overwrite button → `<at-button label={isSaving ? 'Saving...' : 'Overwrite'} onAtuiClick={() => onSave(name.trim())} disabled={isSaving} className="px-4 py-2 text-sm font-medium text-white bg-red-600 rounded-md hover:bg-red-700" />`

Replace the `<input type="text">` for collection name with `<at-input>`:
```tsx
<at-input
  value={name}
  placeholder="Enter a name for this collection"
  onAtuiChange={(e: CustomEvent<string | number>) => setName(String(e.detail))}
  disabled={isSaving}
  className="w-full"
/>
```
Remove the `onKeyDown` handler from the native input (at-input doesn't support it in the same way — remove that UX shortcut).
