**CollectionSelector.tsx — replace native select with at-select:**

The current component uses `<select>` with `<optgroup>` children (Local / Database groups). `at-select` does not support optgroups — flatten into a single options array with labels indicating the group via the option label text.

Adapt the `onChange` call: `at-select` fires `onAtuiChange` with `event.detail` as the selected value string.

Replace the JSX with:
```tsx
'use client';

interface CollectionOption {
  _id: string;
  name: string;
}

interface CollectionSelectorProps {
  collections: CollectionOption[];
  selectedId: string;
  loading: boolean;
  onChange: (id: string) => void;
}

export function CollectionSelector({ collections, selectedId, onChange }: CollectionSelectorProps) {
  const options = [
    { value: 'local', label: 'Local Files' },
    ...collections.map((c) => ({ value: c._id, label: c.name })),
  ];

  return (
    <div className="flex items-center">
      <label className="text-sm font-medium text-gray-700 mr-3">Collection:</label>
      <at-select
        value={selectedId}
        options={options}
        placeholder="Select collection..."
        onAtuiChange={(e: CustomEvent<string>) => onChange(e.detail)}
        className="px-3 py-2 border border-gray-300 rounded-md text-sm bg-white focus:outline-none focus:ring-2 focus:ring-blue-500"
      />
    </div>
  );
}
```

**Approach for at-dialog:**
`at-dialog` is an imperative web component controlled via element methods (`openDialog()`, `closeDialog()`). Use `useRef<HTMLElement>()` for each dialog ref, and call `(ref.current as any).openDialog()` / `(ref.current as any).closeDialog()` in place of `setShow*Modal(true/false)`.

Add three refs at the top of the component:
```typescript
const deleteDialogRef = React.useRef<HTMLElement>(null);
const renameDialogRef = React.useRef<HTMLElement>(null);
const duplicateDialogRef = React.useRef<HTMLElement>(null);
```

Replace all `setShowDeleteModal(true)` → `(deleteDialogRef.current as any)?.openDialog()`
Replace all `setShowDeleteModal(false)` → `(deleteDialogRef.current as any)?.closeDialog()`
Same pattern for rename and duplicate refs.

Remove the `showDeleteModal`, `showRenameModal`, `showDuplicateModal` state variables — dialog open/close is now controlled imperatively via refs.
