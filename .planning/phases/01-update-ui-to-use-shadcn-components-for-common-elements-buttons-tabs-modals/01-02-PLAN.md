Replace with at-tabs using the data-driven `tabs` prop. The `at-tabs` component manages its own active state display — the app still drives `activeTab` via URL, so listen to `onAtuiChange` to call `switchTab`:

```tsx
{/* Tab switcher */}
<div className="bg-white border-b border-gray-200">
  <div className="max-w-7xl mx-auto px-4 sm:px-6 lg:px-8 py-2">
    <at-tabs
      tabs={[
        { id: 'view', title: 'View Tokens' },
        { id: 'generate', title: 'Generate Tokens' },
      ]}
      active_tab={activeTab}
      onAtuiChange={(e: CustomEvent<string>) => switchTab(e.detail as 'view' | 'generate')}
      className="border-none"
    />
  </div>
</div>
```

NOTE: The tab content divs below (with `className={activeTab === 'view' ? '' : 'hidden'}`) stay exactly as-is — the hidden CSS class approach for preserving form state is a locked decision from v1.0. Do NOT replace the content divs with at-tab-content — the current approach must be preserved.
