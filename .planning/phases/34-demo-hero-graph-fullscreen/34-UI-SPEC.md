## Component Inventory

### New Components

**`GraphPanelWithChrome`** (or `TokensGraphSection`)
- Thin wrapper around `TokenGraphPanel`
- Accepts identical props surface as `TokenGraphPanel` today (forwarded unchanged)
- Adds `isFullscreen: boolean` state + `onToggleFullscreen` handler
- Fullscreen layout: `fixed inset-0 z-50 bg-background` (Tailwind utility classes only — no custom CSS)
- Normal layout: passthrough (renders `TokenGraphPanel` as-is, no visual difference)
- Single mounted instance — fullscreen is a CSS layout change, NOT a second mount
- Escape key listener added when fullscreen is active; removed on exit or unmount

**Fullscreen toggle button**
- Placement: graph panel column header area, right side (mirrors the existing `CarouselHorizontal` / `TableSplit` toggle button pattern in `CollectionTokensWorkspace`)
- Size: `h-8 w-8` (32px × 32px), `variant="ghost"`, `size="icon"`
- Icon: Carbon `Maximize` (enter fullscreen) / `Minimize` (exit fullscreen) — or closest Carbon equivalent
- `aria-label="Enter fullscreen graph view"` / `aria-label="Exit fullscreen graph view"`
- `title` tooltip matches aria-label
- No label text — icon only, consistent with existing layout control buttons

### Existing Components (unchanged)
- `TokenGraphPanel` — extracted FROM, not modified beyond accepting wrapper
- `CollectionTokensWorkspace` — wired to use `GraphPanelWithChrome` instead of raw `TokenGraphPanel`
- `GroupStructureGraph` — untouched; key strategy (`{group.id}-{colorThemeId}-{densityThemeId}`) preserved
- shadcn `Button` (`variant="ghost"`, `size="icon"`) — reused from existing imports

### Optional (not required for MVP)
- `localStorage` persistence of fullscreen state under key `tokens-workspace-fullscreen` alongside `tokens-workspace-main-layout`

Source: 34-CONTEXT.md decisions, CollectionTokensWorkspace.tsx lines 107–117, 145–156

## Interaction States

### Fullscreen toggle button

| State | Visual |
|-------|--------|
| Default | `text-muted-foreground`, no background |
| Hover | `hover:bg-accent hover:text-accent-foreground` (shadcn ghost default) |
| Active fullscreen | Icon switches to Minimize; button remains ghost — no filled/active style |
| Focus | `ring-2 ring-ring ring-offset-2` (shadcn focus-visible default) |
| Disabled | Not applicable — always enabled when graph panel is rendered |

### Fullscreen overlay

| State | Visual |
|-------|--------|
| Enter | Instant — no animation. `fixed inset-0 z-50 bg-background`. Graph remounts not triggered. |
| Exit (button) | Instant — layout reverts to prior split/tabs mode |
| Exit (Escape key) | Same as button exit |
| Exit (browser back) | Out of scope for Phase 34 |

### Empty state (no group selected)

| State | Visual |
|-------|--------|
| No group / no token selected | Existing empty state from `TokenGraphPanel` preserved unchanged |
| Fullscreen + no group selected | Same empty state content, now fills full viewport |

## Layout Contract

### Normal mode (split layout)

```
┌─ aside (w-56) ─┬─ ResizablePanel (40%) ─┬─ ResizablePanel (30%) ─┐
│ group tree     │ table / main content   │ GraphPanelWithChrome   │
│                │                        │ [toggle button]        │
└────────────────┴────────────────────────┴────────────────────────┘
```

- Toggle button sits in the graph panel's own column — NOT in the breadcrumb row
- Graph panel border: `border-l border-muted bg-background` (existing, unchanged)

### Fullscreen mode

```
┌─ fixed inset-0 z-50 bg-background ────────────────────────────────┐
│ GraphPanelWithChrome (full viewport)                               │
│ [Minimize toggle button — top-right of graph chrome area]         │
└───────────────────────────────────────────────────────────────────┘
```

- The underlying workspace (table, sidebar) remains mounted but visually hidden by the overlay
- No scroll lock required (graph fills 100% of viewport)
- No backdrop/overlay scrim — background is `bg-background` (opaque)
