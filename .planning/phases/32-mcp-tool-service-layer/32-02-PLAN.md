<objective>
Wire the in-app HTTP API route mutation handlers to the shared service layer, completing MCP-01.

Purpose: The shared services in src/services/shared/ were built to be the single source of truth for token/group/theme business logic. Currently only MCP tools use them. The API routes (tokens, groups, theme tokens) have identical independent MongoDB mutation logic that can silently diverge from the shared implementations. This plan closes that gap by making the API routes delegate their DB mutations to the shared services, keeping auth/billing as route-level wrappers.

Output: Three refactored route files. The token/group/theme mutation logic lives exclusively in src/services/shared/. MCP-01 fully satisfied.
</objective>
