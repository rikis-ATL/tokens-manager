# Phase 32 Plan 01: Extract Shared Service Layer and Add MCP Theme Mutations

**One-liner:** Extract shared token/group/theme business logic from duplicated MCP and in-app implementations into a unified service layer; add theme mutation tools to MCP server for feature parity with in-app AI chat.

## Context and Problem

Currently, we have **two separate implementations** of token/group/theme management logic:

1. **MCP Server** (`src/mcp/tools/`) - Direct MongoDB operations for Claude Desktop/Code
2. **In-App AI Tools** (`src/services/ai/tools.ts`) - HTTP API calls for in-app chat

### Key Issues (MCP-01, MCP-02)

**MCP-01: Duplicate Logic**
- Token CRUD logic exists in both `src/mcp/tools/tokens.ts` (MongoDB) and API routes (HTTP)
- Group operations duplicated between MCP and API handlers
- Theme read operations in MCP but missing theme **mutation** tools

**MCP-02: Missing MCP Theme Tools**
- MCP server has `list_themes`, `get_theme_tokens` (read-only)
- Missing: `create_theme`, `update_theme_token`, `delete_theme_token`, `delete_theme`
- In-app AI chat has full theme mutation capability via HTTP tools
- Claude Desktop/Code users cannot create or modify themes

## Goals

1. **Extract shared service functions** used by both MCP and in-app implementations (MCP-01)
2. **Add theme mutation tools** to MCP server for feature parity (MCP-02)
3. **Ensure identical business logic** between MCP and HTTP handlers

## Plan Overview

### Task 1: Create Shared Service Layer (45 min)

Extract common business logic into `src/services/shared/`:

**`src/services/shared/tokens.ts`**
```typescript
export interface TokenService {
  createToken(collectionId: string, tokenPath: string, value: string, type: string): Promise<ToolResult>
  updateToken(collectionId: string, tokenPath: string, value?: string, type?: string): Promise<ToolResult>
  deleteToken(collectionId: string, tokenPath: string): Promise<ToolResult>
  bulkCreateTokens(collectionId: string, tokens: TokenInput[]): Promise<ToolResult>
}
```

**`src/services/shared/groups.ts`**
```typescript
export interface GroupService {
  createGroup(collectionId: string, groupPath: string, description?: string): Promise<ToolResult>
  renameGroup(collectionId: string, oldPath: string, newPath: string): Promise<ToolResult>
  deleteGroup(collectionId: string, groupPath: string): Promise<ToolResult>
}
```

**`src/services/shared/themes.ts`**
```typescript
export interface ThemeService {
  createTheme(collectionId: string, name: string, colorMode?: string): Promise<ToolResult>
  updateThemeToken(collectionId: string, themeId: string, tokenPath: string, value: string, type?: string): Promise<ToolResult>
  deleteThemeToken(collectionId: string, themeId: string, tokenPath: string): Promise<ToolResult>
  deleteTheme(collectionId: string, themeId: string): Promise<ToolResult>
}
```

### Task 2: Refactor MCP Tools to Use Services (30 min)

Update MCP tools to call shared services instead of direct MongoDB:

- `src/mcp/tools/tokens.ts` → call `TokenService` methods
- `src/mcp/tools/groups.ts` → call `GroupService` methods
- `src/mcp/tools/themes.ts` → call `ThemeService` methods

### Task 3: Add MCP Theme Mutation Tools (30 min)

Create `src/mcp/tools/theme-mutations.ts`:

- `create_theme` - same interface as in-app tool
- `update_theme_token` - same interface as in-app tool
- `delete_theme_token` - same interface as in-app tool
- `delete_theme` - same interface as in-app tool

Update `src/mcp/server.ts` to register theme mutation tools.

### Task 4: Verify Service Equivalence (15 min)

**Audit Check:** Ensure MCP and in-app handlers produce identical database outcomes for the same inputs.

- Create test scenarios for each tool
- Verify MCP service calls match HTTP API behavior
- Document any differences and resolve them
