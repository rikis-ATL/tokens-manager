# Phase 999.1: CSS/HTML pattern token types — Context

**Gathered:** 2026-04-18  
**Status:** Backlog (not yet promoted to a milestone)

<domain>
## Phase boundary

Add **non–W3C-design-token** types used as **CSS/HTML pattern storage** alongside existing tokens. These are **not** official design-token types in the W3C sense; they exist to store reusable CSS/HTML patterns in the same collection model as tokens.

### Type 1: CSS class

- Appears in the **tokens table type** column and as a **new graph node**.
- Stores CSS class–oriented pattern content (exact shape TBD at planning).

### Type 2: HTML template

- **Tokenised HTML** that can reference **CSS class** tokens or **variables** inline.
- Same “pattern storage” role; not an official tokens type.

### Type 3: HTML/CSS component

- **Combination** of CSS + HTML inputs.
- Stored as **string literals in an object format** (shape TBD at planning).
- **Preview:** render in a **minimal third-party sandbox** (e.g. iframe with strict isolation — security and CSP to be defined when promoted).

### Export / Style Dictionary

Official export pipelines may treat these as **out-of-band** or **separate artifacts** — decide when this phase is promoted and REQ-* IDs are added.

</domain>

<problem>
## Problem / motivation (UX)

Linear IDE editing of HTML/CSS/JS makes it hard to see **structure** and **what connects to what** — everything reads as one block. Desirable future directions (not in scope until explicitly planned):

- **HTML:** visually **blocks within blocks**, not only a flat list of `div`s.
- **CSS:** a **navigable, visual style library** (Figma-like) rather than an unrelated list of classes/properties.
- **JS:** remains a separate concern; tying behavior to DOM without constant context switching is painful.

This backlog item captures **storage types + table + graph + sandbox preview** for HTML/CSS patterns. **Full visual editors** (block HTML, Figma-like CSS library) are **deferred** until a promoted phase scopes them.

</problem>

## Future exploration (explicitly out of scope for 999.1 until promoted)

- JS/DOM integration tooling.
