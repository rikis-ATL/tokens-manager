Create `src/components/demo/DemoOverlayCTA.tsx` as a new file:

```typescript
'use client';

import { useSession } from 'next-auth/react';
import { Button } from '@/components/ui/button';
import Link from 'next/link';

/**
 * Overlay CTA displayed over the graph panel for Demo-role users.
 * Shows a "Get started free" sign-up button.
 * Only visible when session.user.role === 'Demo' (D-13).
 * No dismiss mechanism for MVP (D-14).
 * Scaffold for future guided onboarding steps.
 */
export function DemoOverlayCTA() {
  const { data: session } = useSession();
  const isDemoUser = session?.user?.role === 'Demo';

  if (!isDemoUser) return null;

  return (
    <div className="absolute top-3 right-3 z-10">
      <Button asChild size="sm">
        <Link href="/auth/signup">Get started free</Link>
      </Button>
    </div>
  );
}
```

Key constraints:
- `'use client'` at line 1 — `useSession` is client-only.
- Returns `null` when role is not 'Demo' — no DOM output for non-demo users.
- Absolute positioning (top-3 right-3 z-10) — the parent container must have `relative` on it; this is handled in GraphPanelWithChrome (Task 2).
- NO dismiss button (D-14) — the overlay stays visible throughout the hero session.
- NO onboarding steps in this component — scaffold only. Future guided onboarding is deferred.
- `Button asChild size="sm"` with `Link` — this is the project's established CTA pattern from DemoLanding.tsx.
- z-10 keeps the button above the graph canvas but below the fullscreen modal overlay (z-50).
