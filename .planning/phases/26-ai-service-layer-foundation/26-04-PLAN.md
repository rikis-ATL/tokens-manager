The document should be educational — written assuming the reader is learning MCP for the first time. Structure:

**## What is MCP?**
- Model Context Protocol — a standard for connecting AI assistants to external tools and data sources
- Like a USB-C port for AI: any MCP-compatible client (Claude Desktop, Claude Code) can connect to any MCP server
- The server exposes "tools" — functions the AI can discover and call to interact with your application

**## How It Works**
- Stdio transport: the server communicates via stdin/stdout using JSON-RPC 2.0 messages
- The MCP client (Claude Desktop) starts the server process, sends a handshake, discovers available tools, and then calls them as needed during conversation
- Each tool has a JSON Schema (generated from Zod) that tells the AI what parameters it accepts

**## Architecture**
- Diagram showing: Claude Desktop/Code --> stdio --> MCP Server --> Mongoose --> MongoDB
- Explain that the MCP server is a standalone Node.js process that shares the same MongoDB and Mongoose models as the Next.js app
- No running Next.js instance needed — the MCP server connects directly to MongoDB

**## Project Structure**
- `src/mcp/server.ts` — Entry point: creates McpServer, registers tools, connects StdioServerTransport
- `src/mcp/tools/tokens.ts` — Token CRUD tools (list_collections, list_tokens, get_token, create_token, update_token, delete_token)
- `src/mcp/tools/groups.ts` — Group management tools (list_groups, create_group)

**## Running Locally**
```bash
yarn mcp:dev
```
This starts the server on stdio. For manual testing, you typically connect via Claude Desktop or Claude Code — the server is not meant to be used interactively from a terminal.

**## Adding New Tools**
Step-by-step guide:
1. Create a function in the appropriate tools file (or create a new file)
2. Use `server.registerTool("tool_name", { description, inputSchema }, handler)`
3. Define input params with Zod schemas — the MCP SDK generates JSON Schema automatically
4. Return `{ content: [{ type: "text", text: "..." }] }` for success
5. Return `{ content: [{ type: "text", text: "..." }], isError: true }` for tool errors
6. Import and call the registration function in `server.ts`
7. Restart the MCP server (Claude Desktop: toggle the server off/on; Claude Code: restart session)

**## Tool Reference**
Table listing all 8 tools with name, description, and parameters.

**## Important Rules**
- NEVER use `console.log()` — stdout is the JSON-RPC channel
- Always use `console.error()` for debug output
- The server must use `tsconfig.scripts.json` (not the main tsconfig) for `@/` path alias resolution
- MongoDB env vars must be loaded before model imports (handled by `-r dotenv/config`)
