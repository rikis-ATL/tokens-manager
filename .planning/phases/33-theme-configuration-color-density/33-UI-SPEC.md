## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, tight list item insets |
| sm | 8px | Compact element spacing, sidebar item padding horizontal |
| md | 16px | Default element spacing, dialog body padding, header horizontal padding |
| lg | 24px | Section padding, sidebar section header vertical rhythm |
| xl | 32px | Layout gaps between major panels |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: Touch targets on icon-only buttons minimum 32px (`h-8 w-8`). Theme kind badges use 4px vertical + 4px horizontal padding (`py-1 px-1`) to match the existing `ColorModeBadge` pattern in `ThemeList.tsx`.

## Color

Uses the existing shadcn CSS variable system. All values are declared in `globals.css` and respect the `.dark` class toggle.

| Role | CSS Variable | Usage |
|------|-------------|-------|
| Dominant surface (60%) | `--background` / `hsl(var(--background))` | Page background, header, sidebar |
| Secondary surface (30%) | `--card` / `--muted` | Sidebar panels, section dividers, inactive list items |
| Accent — active selection (10%) | `--info` (blue) via `bg-info/10 text-info` | Active color theme row highlight; active density theme row highlight |
| Accent — color mode light badge | `--warning` via `bg-warning/10 text-warning` | Sun/Light badge on color themes only |
| Accent — color mode dark badge | `--muted` via `bg-muted text-muted-foreground` | Moon/Dark badge on color themes only |
| Accent — kind badge (color) | `bg-primary/10 text-primary` | "Color" kind badge on theme rows |
| Accent — kind badge (density) | `bg-secondary text-secondary-foreground` | "Density" kind badge on theme rows |
| Destructive | `--destructive` via `text-destructive` | Delete theme menu item only |
| Border | `--border` / `--muted` | Sidebar section dividers, dialog borders |

Accent reserved for: (1) active theme row highlight, (2) color-mode light/dark badges on color themes, (3) kind badges on all theme rows. Never used for generic hover states or body text.

Source: `globals.css` CSS variables + `ThemeList.tsx` existing badge patterns.

## Focal Point

The dual selector group (Color + Density selectors) in the Tokens page header is the primary focal point for theme context switching. All other theme UI — the sidebar list, dialogs, and graph panel routing — is subordinate to this pair.

## Component Inventory

### New / Modified Components

#### 1. Dual Theme Selector (Tokens page header)

**Location:** `tokens/page.tsx` header — replaces single `<Select>` with a pair of selectors.

**Structure:**
```
[Color: <Select w-36 h-8>] [Density: <Select w-36 h-8>]
```

- Label: `text-xs text-muted-foreground` — "Color:" and "Density:"
- Each `<SelectTrigger>` width: `w-36 h-8 text-sm`
- Default value label: "Default" (maps to no active theme of that kind)
- Only render the Color selector if color themes exist; only render the Density selector if density themes exist. If neither exists, omit the entire group (preserve existing hide-when-empty behavior).
- Selected item in each dropdown shows theme name only (no badge — badges are sidebar-only to keep header compact).

shadcn components: `Select`, `SelectTrigger`, `SelectValue`, `SelectContent`, `SelectItem` — all already in use.

#### 2. ThemeList — Grouped by Kind

**Location:** `src/components/themes/ThemeList.tsx` — restructured into three sections.

**Section order:**
1. **Base** — always present, always first; single non-interactive row with label "Base" and muted text "Collection default tokens". No selection affordance — this is the implicit base layer.
2. **Color themes** — section header + list of color-kind themes + "Add color theme" button.
3. **Density themes** — section header + list of density-kind themes + "Add density theme" button.

**Section header pattern** (reuse existing style):
```
px-3 py-2 border-b border-muted bg-background flex items-center justify-between
  <span class="text-xs font-semibold text-muted-foreground uppercase tracking-wider">{section label}</span>
  <Plus size={14} aria-label="Add color theme" /> button (disabled at limit)
  // Density section uses aria-label="Add density theme"
```

**Theme row pattern** (extend existing):
```
group/item flex items-center pr-1 text-sm cursor-pointer transition-colors
  [active: bg-info/10 text-foreground font-medium]
  [inactive: hover:bg-muted text-foreground]
  <span flex-1 py-1.5 px-3 truncate text-xs>{name}</span>
  <KindBadge />         — color themes: "Color" badge; density themes: "Density" badge
  <ColorModeBadge />    — color themes only: Sun/Light or Moon/Dark badge; NOT shown on density rows
  <DropdownMenu />      — MoreHorizontal trigger aria-label="Theme options", opacity-0 group-hover:opacity-100
```

**Kind badge component (new):**
```tsx
// Color kind
<span className="inline-flex items-center gap-0.5 px-1 py-1 rounded text-[10px] bg-primary/10 text-primary flex-shrink-0">
  <Palette size={9} /> Color
</span>

// Density kind
<span className="inline-flex items-center gap-0.5 px-1 py-1 rounded text-[10px] bg-secondary text-secondary-foreground flex-shrink-0">
  <Layers size={9} /> Density
</span>
```

Lucide icons: `Palette` for color kind, `Layers` for density kind (both already imported in codebase).

**Dropdown menu per row:**
- Color themes: "Switch to Light" / "Switch to Dark" | separator | "Delete"
- Density themes: "Delete" only (no color mode toggle — never shown for density kind)

#### 3. Create Theme Dialog — Extended

**Trigger:** "Add color theme" or "Add density theme" section button.

The dialog `kind` is implicitly determined by which button was pressed — the user does not choose kind inside the dialog.

**Fields:**
- `<Input>` — Theme name (autoFocus, placeholder "Theme name", Enter submits)
- Color mode toggle (Light / Dark buttons) — shown **only when kind === 'color'**. Hidden entirely when creating a density theme.
- Hidden field: `kind` — passed programmatically from the triggering button context.

**Dialog title:**
- "Create Color Theme" when triggered from Color themes section
- "Create Density Theme" when triggered from Density themes section

**Footer:** Cancel (variant="outline") + Create (disabled while name empty)

#### 4. CollectionSidebar — Remove Themes nav item

Remove `{ href: `/collections/${collectionId}/themes`, label: 'Themes', icon: Layers }` from the nav items array. No redirect needed at sidebar level — the old URL `/collections/[id]/themes` should redirect to `/collections/[id]/tokens` at the route level.

#### 5. Config Page — Dual Theme Selector

Replace single `selectedThemeId` `<Select>` with two selectors side-by-side (same pattern as tokens page header):

```
Color theme: [<Select>]   Density theme: [<Select>]
```

Same width/height as header selectors: `w-36 h-8 text-sm`. Same default label: "Default".

## Interaction Contracts

### Theme Selection (Tokens Page Header)

| Action | Result |
|--------|--------|
| Change Color selector to a color theme | `activeColorThemeId` state updates; token table re-renders with color overrides applied; graph panel routes edits to that color theme's `graphState` for color-type groups |
| Change Color selector to "Default" | `activeColorThemeId` = null; collection default color tokens shown |
| Change Density selector to a density theme | `activeDensityThemeId` state updates; token table re-renders with density overrides applied |
| Change Density selector to "Default" | `activeDensityThemeId` = null; collection default dimension tokens shown |
| Both selectors set to non-null | Merged view: collection default → color overrides → density overrides; both selections coexist |

Selection is session-only (D-10 from CONTEXT.md). No persistence to MongoDB.

### Theme List Interactions

| Action | Result |
|--------|--------|
| Click color theme row | Sets `activeColorThemeId`; row highlights with `bg-info/10` |
| Click density theme row | Sets `activeDensityThemeId`; row highlights with `bg-info/10` |
| Click "Add color theme" | Opens Create Color Theme dialog with kind pre-set to `color` |
| Click "Add density theme" | Opens Create Density Theme dialog with kind pre-set to `density` |
| "Add" button at limit (10 themes of kind) | Button disabled with `opacity-40 cursor-not-allowed`; tooltip "Maximum 10 themes per collection" |
| Dropdown → Switch Light/Dark (color theme) | Patches `colorMode` on the theme; badge updates immediately |
| Dropdown → Delete | Opens `AlertDialog` confirmation (see Destructive Actions below) |

### Graph Panel Routing

No visible UI control changes. The graph panel determines its edit target based on the selected group's dominant token type:
- Color-type group → graph edits `activeColorThemeId`'s `graphState` (or collection default if no color theme active)
- Density-type group → graph edits `activeDensityThemeId`'s `graphState` (or collection default if no density theme active)

No toggle or indicator is needed in the graph panel toolbar for this phase.

## States and Edge Cases

### When No Color Themes Exist

The Color themes section still renders (with section header and "Add color theme" button) but the list body shows: `<p class="px-3 py-3 text-xs text-muted-foreground">No color themes yet</p>`.

The Color selector in the header is hidden entirely when zero color themes exist (preserve existing hide-when-empty logic).

### When No Density Themes Exist

Same pattern as above: section renders, Density selector hidden in header.

### Legacy Themes (migration)

After migration, existing themes become `kind: 'color'`. They appear in the Color themes section and retain their `colorMode` badge. No visual disruption.
