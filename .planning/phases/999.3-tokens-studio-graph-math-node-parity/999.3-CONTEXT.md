# Phase 999.3 — Context: Tokens Studio graph math node parity

**Status:** Implementation landed in app; verification and traceability via this backlog phase.

## Problem / opportunity

The composable **Math** node should align with [Tokens Studio Graph Engine — Math](https://documentation.tokens.studio/graph-engine/available-nodes/math) so designers can reason about operations using the same vocabulary and semantics as the reference docs (absolute, variadic add/divide, closest number, fluid scaling, expression “evaluate math” with multiple variables, etc.).

## Reference

- Index: https://documentation.tokens.studio/graph-engine/available-nodes/math  
- Representative pages: Absolute, Add variadic, Divide variadic, Closest number, Fluid, Evaluate Math, Exponentiation (e^x), Difference, Count, Cosine.

## Scope (this phase)

- Extend `MathOp` / `MathConfig`, `graphEvaluator` (`evalMath`), and `MathNode` UI for the agreed operation set.
- Expression mode: variables `a` and `b` in `mathExpression.ts` for **Evaluate Math**–style formulas.
- Jest coverage for new evaluator paths (`graphMathOps.test.ts`, extended `mathExpression.test.ts`).

## Non-goals

- Full parser parity with Tokens Studio (e.g. every function in their engine).
- New React Flow edge types for unlimited fan-in variadic wires; variadic uses a single array port `inputs` plus CSV fallback.

## Dependencies

- None (backlog). Builds on existing graph composable stack (`graph-nodes.types`, `GroupStructureGraph`, `graphEvaluator`).
