## Task breakdown

1. **API — versions (Mongo only)**  
   - `POST` create: validate semver, `buildCollectionSnapshot`, insert `CollectionVersion`, unique `(collectionId, semver)`.  
   - `GET` list: metadata only.  
   - `GET` `[versionId]`: include `snapshot`.  
   - `POST` restore: `ManageVersions`, block or warn `isPlayground`, `repo.update` with snapshot fields only.

2. **API — collection NPM fields**  
   - Extend `PUT /api/collections/[id]`: optional `npmToken` → encrypt + `$set` / clear; `npmPackageName`, `npmRegistryUrl`.  
   - `GET` collection: include `npmPackageName`, `npmRegistryUrl`, `npmTokenConfigured`, `themes`, `colorFormat`, `accentColor` as needed for settings.

3. **Service — `npm-publish.service.ts`**  
   - Resolve `ICollectionSnapshot` from live doc or version doc.  
   - For each theme + default: `mergeThemeTokens` → `buildTokens` → write `dist/<slug>/tokens.{css,json,js}`.  
   - `tar.create` → `libnpmpublish.publish` with `opts.token`, `opts.registry`.

4. **GUI**  
   - Settings: NPM package name, registry URL, token (masked save).  
   - Versions page: save (semver + note), table, restore confirm, publish dialog (source live vs version, semver).

5. **Supabase**  
   - `rowToDoc`: default `npmPackageName`, `npmRegistryUrl`, `npmTokenConfigured: false`, `accentColor` if missing.
