# Phase 999.2: Versioning and NPM publish — Context

**Status:** Backlog (captured for future planning / parallel implementation track)

## Domain

Immutable **collection versions** (semver + full design snapshot: default tokens, `graphState`, all **themes**) stored in MongoDB `collectionversions`. **NPM publish** packages per-theme Style Dictionary outputs (`mergeThemeTokens` + `buildTokens`) as a tarball via `libnpmpublish`, using per-collection encrypted NPM token and configurable registry (npmjs or GitHub Packages). All flows are GUI-driven.

## Decisions (when promoted)

- Snapshots exclude integration secrets (Figma/GitHub/NPM tokens); restore applies design fields only.
- MongoDB-only for versions and encrypted NPM token (Supabase: 501 for these features until schema parity).
- Permissions: `ManageVersions`, `PublishNpm` (Editor+Admin); Demo mode allows same as GitHub push when `DEMO_MODE`.
