  <what-built>
    Complete multi-row selection and bulk actions for the token table:
    - Always-visible checkbox column (leftmost, ~40px)
    - Header checkbox for select-all / deselect-all
    - Shift-click range selection
    - Floating action bar above the table when rows are selected (shows "N selected", Escape to dismiss)
    - Bulk Delete with confirmation dialog
    - Bulk Move to group via tree-picker modal
    - Bulk Change Type via type select
    - Bulk Add Prefix with inline expand and live preview
    - Bulk Remove Prefix with auto-detected common prefix pre-fill and live preview
    - Undo stack integration (Ctrl+Z undoes last bulk action)
    - Source-mode hides checkboxes
    - Group switch clears selection
    - Works in Default theme and custom theme Enabled groups
  </what-built>
  <how-to-verify>
    Start the dev server (`yarn dev`) and open a collection with at least one group containing several tokens.

    **Scenario 1 — Checkbox column visible**
    - Open a token group. Confirm checkboxes are visible in the leftmost column of every row, no hover required.
    - Confirm the header row also has a checkbox.

    **Scenario 2 — Select all / deselect all**
    - Click the header checkbox → all row checkboxes become checked. Action bar appears with "N selected".
    - Click header checkbox again → all deselected. Action bar disappears.

    **Scenario 3 — Shift-click range**
    - Click row 1's checkbox. Click row 4's checkbox while holding Shift.
    - Rows 1–4 are all selected (selected rows show blue-50 highlight with ring).

    **Scenario 4 — Escape clears selection**
    - With rows selected, press Escape. Action bar disappears, all checkboxes unchecked.

    **Scenario 5 — Bulk delete**
    - Select 2–3 tokens. Click Delete in the action bar.
    - Confirm dialog appears showing the correct count.
    - Click Delete. Tokens are removed from the group.
    - Press Ctrl+Z (or Cmd+Z). Tokens are restored.

    **Scenario 6 — Bulk move to group**
    - Select 2 tokens. Click Move in the action bar.
    - Group tree picker modal opens. Confirm it shows the full group tree with indentation.
    - The source group is visually disabled (grayed out, not clickable).
    - Click a destination group. Modal closes. Tokens appear in the destination group.
    - Press Ctrl+Z. Tokens are back in the source group.

    **Scenario 7 — Bulk change type**
    - Select 2 tokens. Click the "Change type" select in the action bar.
    - Choose a different type. Both tokens update to the new type.
    - Press Ctrl+Z. Types are restored.

    **Scenario 8 — Add prefix with live preview**
    - Select 2–3 tokens. Click "Add prefix" in the action bar.
    - An inline text input expands. Type "brand-".
    - Live preview shows the renamed paths (up to 5) in monospace below.
    - Click Apply. Token paths are renamed with the prefix.
    - Press Ctrl+Z. Paths are restored.

    **Scenario 9 — Remove prefix with auto-detect**
    - Select tokens that share a common prefix (e.g. "color-red", "color-blue").
    - Click "Remove prefix". The input is pre-filled with "color-" (auto-detected).
    - Live preview shows renamed paths.
    - Click Apply. Prefix is removed.

    **Scenario 10 — Source mode hides checkboxes**
    - Activate a custom theme. Navigate to a group in Source state.
    - Confirm: no checkbox column visible, no action bar.

    **Scenario 11 — Group switch clears selection**
    - Select tokens in group A. Click a different group in the sidebar.
    - Return to group A. Confirm: no tokens selected, action bar gone.

    **Scenario 12 — Works in custom theme Enabled group**
    - Activate a custom theme. Navigate to a group in Enabled state.
    - Select tokens. Run a bulk action (e.g. change type).
    - Confirm the change is saved to the theme (not the master collection).
    - Press Ctrl+Z. Change is undone.
  </how-to-verify>
  <resume-signal>Type "approved" if all 12 scenarios pass, or describe any issues found.</resume-signal>
