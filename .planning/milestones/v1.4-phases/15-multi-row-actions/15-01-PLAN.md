  <behavior>
    Six pure functions operating on `TokenGroup[]` trees. All inputs/outputs are serialisable data — no React, no side-effects.

    **resolveTokenPathConflict(existingPaths: Set&lt;string&gt;, candidate: string): string**
    - If `candidate` not in `existingPaths` → return `candidate` unchanged
    - Try `${candidate}-2` through `${candidate}-10`; return first that is not in set
    - Fallback: `${candidate}-${Date.now()}`

    **detectCommonPrefix(paths: string[]): string**
    - Empty or single-element array → return `''`
    - Sort array; compare first vs last character-by-character; cut at last `-` or `.` boundary
    - Example: `['color-red', 'color-blue', 'color-green']` → `'color-'`
    - Example: `['red', 'blue']` → `''` (no common prefix)

    **bulkDeleteTokens(groups: TokenGroup[], groupId: string, tokenIds: Set&lt;string&gt;): TokenGroup[]**
    - Find the group with matching `id` (recursive)
    - Return new tree with that group's `tokens` filtered to exclude matching IDs
    - All other groups/tokens unchanged

    **bulkMoveTokens(groups: TokenGroup[], sourceGroupId: string, destGroupId: string, tokenIds: Set&lt;string&gt;): TokenGroup[]**
    - Remove `tokenIds` from source group's tokens
    - Append them to destination group's tokens, resolving path collisions with `resolveTokenPathConflict` against existing dest token paths
    - Moving to same group → no-op (return groups unchanged)
    - Non-existent source or dest group → return groups unchanged

    **bulkChangeType(groups: TokenGroup[], groupId: string, tokenIds: Set&lt;string&gt;, newType: TokenType): TokenGroup[]**
    - Find the group; return new tree with matching tokens having `type: newType`, all other fields unchanged

    **bulkAddPrefix(groups: TokenGroup[], groupId: string, tokenIds: Set&lt;string&gt;, prefix: string): TokenGroup[]**
    - For each selected token: rename `token.path` to `${prefix}${token.path}`, resolve collisions
    - After renaming, scan ALL tokens in the group: if any token's `value` is an alias string like `{groupName.oldPath}` referencing a renamed path, update it to `{groupName.newPath}` (rewrite within-group aliases only; cross-group alias rewriting is out of scope)
    - Empty prefix → return groups unchanged

    **bulkRemovePrefix(groups: TokenGroup[], groupId: string, tokenIds: Set&lt;string&gt;, prefix: string): TokenGroup[]**
    - For each selected token: if `token.path.startsWith(prefix)` → rename to `token.path.slice(prefix.length)`, resolve collisions; otherwise skip silently
    - After renaming, rewrite within-group aliases referencing renamed paths (same as addPrefix)
    - Empty prefix → return groups unchanged

    Cases:
    - bulkDeleteTokens(groups, 'g1', new Set(['t1','t2'])) → group g1 tokens without t1,t2
    - bulkDeleteTokens(groups, 'g1', new Set()) → groups unchanged
    - detectCommonPrefix(['color-red','color-blue']) → 'color-'
    - detectCommonPrefix(['abc','xyz']) → ''
    - detectCommonPrefix([]) → ''
    - bulkMoveTokens with collision → destination gets `tokenPath-2`
    - bulkAddPrefix with alias rewrite: token value `{myGroup.old-name}` → `{myGroup.pfx-old-name}`
    - bulkRemovePrefix skips token whose path does not start with prefix
  </behavior>
