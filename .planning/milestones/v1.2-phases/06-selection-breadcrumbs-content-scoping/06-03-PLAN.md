    This filter only checks top-level `tokenGroups`. For groups that live as children deep in the tree, this filter will find nothing (empty array) when selectedGroupId is a nested group's ID.

    **Content scoping for nested groups:** The `selectedGroupId` is the ID of the selected node anywhere in the tree. `TokenGeneratorFormNew` receives this prop but filters only top-level groups. To scope to a nested group, the page needs to pass the correctly-resolved group to the form.

    Add a computed value in the tokens page using the existing `findGroupById` utility:
    ```tsx
    import { findGroupById } from '@/utils';
    ```
    Then compute:
    ```tsx
    const selectedGroup = selectedGroupId
      ? findGroupById(masterGroups, selectedGroupId) ?? null
      : null;
    ```
    Check if `findGroupById` is already exported from `@/utils` (it exists in `src/utils/token.utils.ts` — verify the barrel export in `src/utils/index.ts` exports it).

    **IMPORTANT:** If `findGroupById` is NOT exported from `src/utils/index.ts`, add the export: look at `src/utils/index.ts` and add `export { findGroupById } from './token.utils'` or `export * from './token.utils'` if not already present.

    Then pass `selectedGroup?.tokens` directly to the form, or — the simpler approach — let the existing `selectedGroupId` prop in `TokenGeneratorFormNew` handle it. The current filter `tokenGroups.filter(g => g.id === selectedGroupId)` only handles top-level. For a truly nested scenario we need the form to find the group recursively.

    **Pragmatic approach for this phase:** Pass `selectedGroupId` as-is (already wired). The form's filter `tokenGroups.filter(g => g.id === selectedGroupId)` will correctly show 0 groups for nested nodes (since they live in `children`, not in top-level `tokenGroups`). To handle this:

    In `TokenGeneratorFormNew.tsx`, update the rendering filter from:
    ```tsx
    (selectedGroupId
      ? tokenGroups.filter(g => g.id === selectedGroupId)
      : tokenGroups
    ).map(group => renderGroup(group))
    ```
    to use a recursive find:
    ```tsx
    (() => {
      if (!selectedGroupId) return tokenGroups;
      // Try top-level first (fast path)
      const topLevel = tokenGroups.find(g => g.id === selectedGroupId);
      if (topLevel) return [topLevel];
      // Find anywhere in tree
      const found = findGroupById(tokenGroups, selectedGroupId);
      return found ? [found] : tokenGroups;
    })().map(group => renderGroup(group))
    ```

    Add `import { findGroupById } from '../utils';` to TokenGeneratorFormNew if not already present (check existing imports at top of file — it imports from `'../utils'` already).

    **4. Add "No tokens in this group" empty state in TokenGeneratorFormNew:**
    After the `renderGroup` map (where 0 groups or a group with 0 tokens would render an empty area), add:

    ```tsx
    {selectedGroupId && (() => {

      return found && found.tokens.length === 0 && (!found.children || found.children.length === 0 || true)
        ? <div className="p-6 text-sm text-gray-400 text-center">No tokens in this group</div>
        : null;
    })()}
    ```

    Simpler: after the `.map(group => renderGroup(group))` line, add a conditional that checks if the resolved group has 0 direct tokens:
    ```tsx
    {selectedGroupId && (() => {
      const found = findGroupById(tokenGroups, selectedGroupId)
                 ?? tokenGroups.find(g => g.id === selectedGroupId);
      if (!found) return null;
      if (found.tokens.length === 0) {
        return <p className="p-6 text-sm text-gray-400 text-center">No tokens in this group</p>;
      }
      return null;
    })()}
    ```

    This handles the "parent-only group" case per user decision.
