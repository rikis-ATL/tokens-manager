  <action>
Create two new route files following existing API patterns (requireRole guard, broadcastTokenUpdate, lean queries).

**src/app/api/collections/[id]/tokens/route.ts** — three handlers for single-token operations on collection default tokens:

1. `POST` — Create a single token. Body: `{ tokenPath: string, value: string, type?: string }`. Uses `requireRole(Action.Write, params.id)`. Uses `TokenCollection.findByIdAndUpdate(params.id, { $set: { ["tokens."+tokenPath+".$value"]: value, ["tokens."+tokenPath+".$type"]: type || "color" } }, { new: true })`. Returns `{ success: true, token: { path, $value, $type } }`. Calls `broadcastTokenUpdate(params.id)` after mutation. Returns 404 if collection not found. Returns 400 if `tokenPath` missing.

2. `PATCH` — Update an existing token's value/type. Body: `{ tokenPath: string, value?: string, type?: string }`. Same guard and pattern as POST but only sets provided fields. Returns 400 if neither value nor type provided.

3. `DELETE` — Delete a token by path. Body: `{ tokenPath: string }`. Uses `$unset: { ["tokens."+tokenPath]: "" }`. Returns `{ success: true }`. Calls `broadcastTokenUpdate(params.id)`.

All three handlers: import `requireRole`, `Action`, `broadcastTokenUpdate`, `TokenCollection`, `dbConnect`. Start with `const authResult = await requireRole(Action.Write, params.id); if (authResult instanceof NextResponse) return authResult;` then `await dbConnect();`.

**src/app/api/collections/[id]/groups/route.ts** — three handlers for group operations:

1. `POST` — Create a group. Body: `{ groupPath: string, description?: string }`. Uses `$set: { ["tokens."+groupPath]: description ? { "$description": description } : {} }`. Returns 400 if groupPath missing. Idempotent if group exists.

2. `PATCH` — Rename a group. Body: `{ oldPath: string, newPath: string }`. Implementation: fetch collection, navigate to `tokens[oldPath]` in the tokens object, set `tokens[newPath]` to the value, `$unset` the old path, save using `findByIdAndUpdate` with both `$set` and `$unset` in one operation:
   ```
   await TokenCollection.findByIdAndUpdate(params.id, {
     $set: { [`tokens.${newPath}`]: groupValue },
     $unset: { [`tokens.${oldPath}`]: "" }
   });
   ```
   This atomically moves all tokens under the old path to the new path. Returns 400 if oldPath/newPath missing. Returns 404 if group not found.

3. `DELETE` — Delete a group and all its tokens. Body: `{ groupPath: string }`. Uses `$unset: { ["tokens."+groupPath]: "" }`. Calls `broadcastTokenUpdate(params.id)`.

All group handlers also use `requireRole(Action.Write, params.id)` and call `broadcastTokenUpdate(params.id)` after mutations.

Per D-10, these endpoints wrap the same Mongoose logic as the MCP tools but go through the standard Next.js route handler pattern with auth, per AI-15.

The `themeId` param is passed through for future theme-aware operations but Phase 28 tool calls target `collection.tokens`. Note this in a code comment.
   - Dispatches to the appropriate endpoint:
